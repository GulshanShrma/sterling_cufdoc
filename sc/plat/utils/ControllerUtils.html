ControllerUtils

This utility class contains methods related to the controllers.

Methods

getMashupContext
The getMashupContext method returns a new instance of mashupContext object. This object is used to call any mashup from the UI.
Arguments
NameTypeDescription
screenScreenA screen instance for which mashup or mashups will be called
Returns : Bean

setMashupContextIdenitier
The setMashupContextIdenitier method sets the unique string identifier for the mashup call. Normally, mashupref fulfills the role of a unique identifier. However, there are scenarios when the same mashup gets called twice and we need a way to identify the uniqueness of the calls. This method is used in such scenarios to set the unique identifier value.
Arguments
NameTypeDescription
mashupContextBeanMashupContext for a mashup call
identifierStringUnique identifier for a mashup context

openScreenInEditor
The openScreenInEditor method opens a screen in editor.
Arguments
NameTypeDescription
screenIdStringID of a screen to open in editor
screenInputModelInitial input for a screen. Also called as editor input
oldScreenScreenOptional argument with the old screen instance
constructorArgsBeanOptional argument with the constructor arguments for the new screen
optionsBeanOptional argument with any additional options needed to open a screen
editorIdStringUnique ID of the editor

continueOpeningInEditor
The continueOpeningInEditor method continues to open the screen/wizard in the editor using the data passed as input. It should not be called from any place other than inside the 'beforeOpenScreenInEditor' method of the editor.
Arguments
NameTypeDescription
dataBeanSame data object passed to the beforeOpenScreenInEditor method of the editor
