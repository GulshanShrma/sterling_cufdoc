PaginationUtils

This utility class contains methods related to pagination in grids.

Methods

getInitialInputForPaginatedGrid
The getInitialInputForPaginatedGrid method returns the inputJson parameter passed to the grid when pagination was started through startPagination method. This attribute is fetched from the grid store and the application must not modify the returned value. It must be treated as read-only.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
Returns : Model

getCurrentPageLastRecordForPaginatedGrid
The getCurrentPageLastRecordForPaginatedGrid method returns the last record from the current page of a paginated grid.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
Returns : Model

getLastRecordMapForPaginatedGrid
The getLastRecordMapForPaginatedGrid method returns the cached last record map of a paginated grid.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
Returns : Model

getPagingConfigForGrid
The getPagingConfigForGrid method returns the _pagingConfig attribute that is stored on a paginated grid. This attribute is fetched from the grid store and the application must not modify the returned value. It must be treated as read-only.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
Returns : Model

getPageSizeForPaginatedGrid
The getPageSizeForPaginatedGrid method returns the page size of a paginated grid. When page size is changed from the UI, it is advisable to use the event handler of the beforePageSizeChange event to get the old page size or the new page size.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
Returns : Number

getCurrentPageNumberForPaginatedGrid
The getCurrentPageNumberForPaginatedGrid method returns the page number of the current page of a paginated grid.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
Returns : Number

getPaginationDataFromMashupContext
The getPaginationDataFromMashupContext method returns the output for a page from a paginated mashup call. Returns null if any of the parameters are invalid or null.
Arguments
NameTypeDescription
mashupContextBeanThe same mashupContext object that is present as an argument to the handleMashupCompletion method
reqdMashupRefIdStringThe ID of the mashup for which the output data should be fetched
Returns : Model

getMashupContextForPaginatedBehaviorCall
The getMashupContextForPaginatedBehaviorCall method returns the mashupContext object that can be used to invoke mashup in case of a paginated grid. Returns null if screenInstance or gridxUId is null.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
pageSizeStringNumber of records to be fetched for a page. Defaults to the current pageSize value in the grid store. If that is also not set, defaults to 5
pageNoNumberThe page number to load through the API call. Defaults to the current pageNo value on the grid store. If that is also not set, defaults to 1
mashupRefIdStringOptional argument indicating the ID of the mashup for which the paginated call is to be made. Defaults to the one in the grid store. If the grid store does not have this information, null is returned
mashupContextBeanOptional argument if passed will override any pagination specific attributes. Other attributes will be retained
maskUIBooleanOptional argument to display mask on UI during the API call. Defaults to true. NOTE: This attribute is completely ignored if mashupContext is passed
Returns : Bean

getMashupRefObjectForPaginatedBehaviorCall
The getMashupRefObjectForPaginatedBehaviorCall method returns a mashupRefObject that can be added to the mashupRefList array to invoke mashup for a paginated grid. Returns null if the following conditions are not met: Either screenInstance or gridxUId is null OR mashupRefId or originalInput was not passed or could not be calculated.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
mashupRefIdStringThe mashup reference id of the mashup call. OPTIONAL only if data has been set on the grid at least once prior to this method call
originalInputModelThe original input data to the grid. NOTE: 1. If the data is being set on the grid for the first time or input data has not been set on the grid using other utility methods, this attribute becomes MANDATORY. 2. If data has been set on grid at least once or a utility has been called to set the input data then this attribute is OPTIONAL
Returns : Model

startPaginationUsingUId
The startPaginationUsingUId method starts the pagination on a grid. Use this method when pagination should be started by a behavior call or when the Mashup call should be separate and be made by the grid itself.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
inputJsonModelThe input data to the grid to start pagination. This is the input to the pagination API
optionsBeanOther options that can be used for pagination like the following attributes: mashupRefId, screen

loadPageFromMashupContext
The loadPageFromMashupContext method loads the required page or changes pagination size based on data in the mashupContext object. Appropriate utility methods are invoked automatically. Returns true if the page was loaded or false otherwise.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
originalInputModelThe mashupContext object that was possibly prepared using getMashupContextForPaginatedBehaviorCall method. This object is also present as an argument in handleMashupCompletion method
mashupRefIdStringThe mashup reference ID for which data needs to be loaded

getPaginationUIActionFromMashupContext
The getPaginationUIActionFromMashupContext method returns the paginationUIAction string from the paginatedMashupRefIdArray defined in the mashupContext.
Arguments
NameTypeDescription
mashupContextBeanThe mashupContext object that is present as an argument to the handleMashupCompletion method
reqdMashupRefIdStringThe mashup reference ID for which the paginationUIAction needs to be fetched
Returns : String

loadNextPage
The loadNextPage method loads the next page data on the grid. Returns true if data is loaded successfully or false otherwise. NOTE: 1. Assumption is that UI event has already been triggered by user action. This will not invoke the beforeNextPage event. It will trigger beforeGotoPage event. 2. Method should be used when paginationActionsCustomImpl attribute is set as true in pagination binding.
Arguments
NameTypeDescription
screenInstanceScreenThe instance of the screen that contains the grid
gridxUIdStringThe unique identifier of the grid
paginatedApiOutputModelThe output of a paginated API call. mashupContext is an argument to the handleMashupCompletion method. paginatedApiOutput can be obtained from it by calling the getPaginationDataFromMashupContext utility method
originalInputModelOptional argument that refers to the original input data to the grid. Present only for backward compatibility
optionsBeanOptional argument that refers to the additional options needed for pagination. Present only for backward compatibility
Returns : Boolean

Assumption is that UI event has already been triggered by user action. This will not invoke the beforePreviousPage event. It will trigger beforeGotoPage event. 2. Method should be used when paginationActionsCustomImpl attribute is set as true in pagination binding.
Arguments

originalInputModelOptional argument that refers to the original input data to the grid. Present only for backward compatibility
optionsBeanOptional argument that refers to the additional options needed for pagination. Present only for backward compatibility
Returns : Boolean

changePageSize
The changePageSize method changes the pagination page size. NOTE: 1. Assumption is that UI event has already been triggered by user action. This will not invoke the beforePageSizeChange event. 2. Method should be used when paginationActionsCustomImpl attribute is set as true in pagination binding.
Arguments

originalInputModelOptional argument that refers to the original input data to the grid. Present only for backward compatibility
optionsBeanOptional argument that refers to the additional options needed for pagination. Present only for backward compatibility

startPaginationWithOutputData
The startPaginationWithOutputData method starts the grid pagination with the first page data already available to paint on UI.
Arguments
NameTypeDescription
screenScreenThe instance of the screen that contains the grid
uIdStringThe unique identifier of the grid
inputDataModelThe input data needed for grid pagination
outputDataModelThe first page output data of the paginated grid
optionsBeanThis object is mandatory and it should have mashup reference ID for the behavior call and screen in which the mashup reference is defined

hidePaginationBar
The hidePaginationBar method hides the pagination bar on a paginated grid.
Arguments
NameTypeDescription

gridxUIdStringThe unique identifier of the grid

showPaginationBar
The showPaginationBar method shows the pagination bar on a paginated grid.
Arguments

gridxUIdStringThe unique identifier of the grid
skipOwnerScreenResizeBooleanOptional argument if passed as true will skip resizing the owner screen
