ScreenUtils

This utility class contains methods related to the current screen.

Methods

setModel
The setModel method sets the passed model data on the screen for a given namespace.
Arguments
NameTypeDescription
screenScreenA screen instance
namespaceStringA namespace of the above screen instance
modelDataModelJSON object to populate data on the screen instance
optionsBeanOptional argument passed as an object with the following property: clearOldVals : Passed as true to remove existing values from widgets which are not populated using modelData above (Defaults to false)

getModel
The getModel method returns the model for a given namespace which was set using setModel method.
Arguments
NameTypeDescription
screenScreenA screen instance
namespaceStringA namespace of the above screen instance
Returns : Model

getTargetModel
The getTargetModel method returns the model object from the screen for a given namespace.
Arguments
NameTypeDescription
screenScreenA screen instance
namespaceStringA namespace of the above screen instance
optionsBeanOptional argument to specify the following option as an object. For grid you can pass modified|deleted|added flag in options object if either one of them is true (or all are true). getTargetModel method returns only those records which have been modified|deleted|added. By default these flags are false. Also allowEmpty flag can be passed to return attributes with no data (Defaults to false)
Returns : Model

ignoreStaticBindings
The ignoreStaticBindings method adds the list of static bindings to the existing list of ignored static bindings on the screen (The ignored static bindings are not used when taget model is returned).
Arguments
NameTypeDescription
screenScreenA screen instance
staticBindingsListList of static binding objects containing source and target bindings

getString
The getString method returns a string from the localized bundles.
Arguments
NameTypeDescription

keyStringThe bundle key to search for in the localized bundles
Returns : String

The getFormattedString method returns a formatted string from the localized bundles.
Arguments
NameTypeDescription

argsALLThe object or array of strings that should be substituted in context sensitive value. The args will take a Bean, Model or List
Returns : String

The getLabelString method returns a label string with a colon added to it.
Arguments
NameTypeDescription
screenScreenA screen instance

Returns : String

showErrorMessageBox
The showErrorMessageBox method opens an error message box on the screen.
Arguments
NameTypeDescription

Arguments

validate
The validate method validates if every widget on the screen has a valid value or is in error.
Arguments
NameTypeDescription
screenScreenA screen instance
Returns : Boolean

isValid
The isValid method validates if any of the screen widgets that are associated with the namespace passed contain any errors. If namespace is not passed, then all namespaces will be considered.
Arguments
NameTypeDescription
screenScreenA screen instance
namespaceStringOptional
Returns : Boolean

isDirty
The isDirty method tests if the passed namespace is dirty(changed from previous saved state) or not and returns a boolean accordingly. If namespace is not passed the check is done for all the screen namespaces.To do a dirty check for multiple namespaces, this method has to be called once for each namespace.
Arguments
NameTypeDescription
screenScreenA screen instance
namespaceStringOptional
forceCheckBooleanOptional argument passed as true to force the dirty state check again for all the widgets(Defaults to false)
Returns : Boolean

focusFirstEditableWidget
The focusFirstEditableWidget method sets the focus on the first editable control on the screen.
Arguments
NameTypeDescription
screenScreenA screen instance

clearScreen
The clearScreen method resets all the dirty states of the screen to clean.
Arguments
NameTypeDescription
screenScreenA screen instance
namespaceStringOptional argument with the namespace value, to clear the dirty state of all the widgets associated with that namespace. If namespace is not passed, then all the widgets associated with all the namespaces will be marked clean
skipParentDirtyRecalculationBooleanOptional argument with value as true to revert to old behavior for repeating screens when a single screen becoming clean would mark parent clean without checking the current state of other repeating screens

getInitialInputData
The getInitialInputData method returns the initial input data passed to screen.
Arguments
NameTypeDescription

Returns : Model

Arguments

scrollToTop

Arguments
NameTypeDescription

disableFieldsInChildScreen
The disableFieldsInChildScreen method enables or disables all the controls in a child screen based on the uId of the child screen.
Arguments
NameTypeDescription
screenScreenThe parent screen object that contains the screen that needs to be enabled or disabled
childIdStringThe uId for the screen object that needs to be enabled or disabled
paramsBeanparams.disable can be set to true/false to disable/enable the child screen instance respectively

showChildScreen
The showChildScreen method shows the child screen at a later stage for a dynamic screen reference. If the child screen is already rendered, the show method is called for the same. If not, based upon the options provided, the init API is called/not called for the child screen during the rendering process.
Arguments
NameTypeDescription
screenScreenScreen instance of the parent in which the new screen has to be opened
screenRefUIdStringScreen ID of the child screen
constructorArgsBeanConstructor args for the the child screen
callbackhandlerStringFunction on the parent screen which will be invoked once this method finishes execution
optionsBeanOptions related to the new screen. shouldCallInitApi (Boolean, if true will cause init API call to be called. Otherwise init API will be skipped)
screenInputModelScreen input model
